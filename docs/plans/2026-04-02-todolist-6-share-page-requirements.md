# TODOLIST-6: 分享页需求与信息架构

本文档明确 **分享页（只读对外展示）** 的目标、页面信息架构、URL 规则与权限边界，作为后续实现与评审的依据。与现有代码域模型对齐：`Task`（嵌套 `subtasks`、`dependsOn`）、`TaskEngine` 的完成规则与统计（见 `README.md`、`src/lib/types.ts`、`src/lib/statistics.ts`）。

---

## 1. 目标与范围

**目标：** 允许用户生成一个 **只读链接**，使未登录访客在浏览器中查看某次 **任务图快照**（根任务列表、层级、依赖关系、状态），用于协作对齐、进度同步或归档展示。

**MVP 范围内**

- 分享页 **不** 提供完成/编辑/增删任务；所有交互限于导航与展示（可含「复制链接」等元操作）。
- 展示内容与主应用 **同一套语义**：`TaskStatus`、`dependsOn` / blocked-by、`listDependents` 的只读呈现。

**明确不在 MVP（可列为后续）**

- 访客评论、实时协作、细粒度「谁能看」账号体系。
- 分享页内修改数据后回写所有者会话（若未来需要，应单独立项）。

---

## 2. 信息架构（页面结构与展示内容）

### 2.1 页面区块（自上而下）

| 区块 | 内容 | 说明 |
|------|------|------|
| **页眉** | 产品名或列表标题、可选「快照时间」 | 若后端存储快照，显示 `asOf` ISO 时间；纯前端方案可省略或显示「生成于本地会话」类说明。 |
| **摘要** | 可选一行说明 | 例如所有者填写的「给查看者的说明」（若有）。 |
| **核心：任务树** | 与主界面同构的树形列表 | 每项至少：**标题**、**状态**（`pending` / `in_progress` / `completed` / `cancelled`）。 |
| **任务元信息（只读）** | 与 `todo-app` 一致的依赖信息 | **Blocked by**：`dependsOn` 解析为前置任务标题或 ID；**Dependents**：`listDependents` 只读列表。不展示可点的「Complete」作为主操作。 |
| **可选：统计条** | 与 `computeTaskStatistics` 对齐的摘要 | 总任务数、根任务数、最大深度、前置边数量、按状态计数等；用于一眼扫全局。 |
| **页脚** | 返回主站 / 帮助 / 隐私声明链接 | 按产品规范配置。 |

### 2.2 展示内容规则

- **层级：** 按 `subtasks` 递归展开，深度与主应用一致；过长列表允许折叠（实现阶段再定默认展开层级）。
- **状态：** 仅用展示组件（徽章/文案），不改变状态。
- **依赖：** 图中不存在「环」时展示边；若快照含历史非法状态，展示策略与引擎一致（引擎侧应已拒绝环；分享数据若来自导出，可做校验失败页）。
- **敏感信息：** 任务 `title` 视为用户可控文本；分享即表示用户同意公开这些内容。不在分享页展示本地存储键、用户账号标识（除非未来显式设计「署名」字段）。

## 3. URL 规则

### 3.1 路径形态（建议）

采用 **不透明 token** 作为路径参数，避免与业务 `task.id` 混用、避免枚举：

- **推荐：** `/share/[token]`
  - `token`：高熵随机字符串（例如 URL-safe Base64 或 `nanoid` 风格），长度建议 ≥ 16 字符有效熵（实现时按安全标准取值）。

**保留扩展（非 MVP 必做）**

- 查询参数：`?v=1` 用于 **快照版本** 或 API 版本，仅在需要破坏缓存或兼容旧链时使用。
- 不采用 `/task/[id]/share` 作为主形态，以免暴露可猜测的内部 ID；若需「锚点到某任务」，可在分享页内用 hash：`/share/[token]#task-[id]`（仅滚动定位，不改变权限）。

### 3.2 规范与 SEO

- **HTTP：** 对外分享一律 **HTTPS**。
- **Canonical：** 每个 `token` 对应唯一 canonical URL（无尾斜杠或全站统一一种，与 Next.js `trailingSlash` 一致）。
- **元数据：** `title` / `description` 可由快照中的列表标题与任务数生成；无快照标题时使用通用文案。

### 3.3 失效与错误页

| 情况 | 行为 |
|------|------|
| `token` 不存在或已撤销 | `404` 或专用「链接已失效」页（不泄露是否存在过该 token）。 |
| 后端暂不可达 | `503` 或友好重试提示（若未来有 API）。 |

## 4. 权限与安全模型

### 4.1 角色（逻辑）

| 角色 | 能力 |
|------|------|
| **链接持有者（访客）** | 仅能通过有效 URL **读取**该次快照允许展示的数据；不能改任务、不能列出他人其他分享。 |
| **所有者（应用内登录用户，若未来有账号）** | 创建分享、撤销、（可选）设置过期时间、查看「当前有效链接」列表。 |
| **系统** | 生成/校验 token、存储快照、审计（若合规需要）。 |

**当前仓库无服务端持久化：** MVP 若先做「仅复制序列化 JSON + 数据 URL / 第三方托管」则权限模型退化为 **任何拿到载荷的人可读**；一旦上线 **托管式分享链接**，必须由服务端校验 `token` 并只返回绑定快照。

### 4.2 机密性与猜测难度

- `token` 必须 **不可预测**；禁止自增数字 ID 作为唯一凭证。
- 撤销分享后，旧 URL 必须立即失效（服务端删除或标记失效）。

### 4.3 与「统计页」关系

- `/stats` 类页面若为 **全局 demo**，与 **按 token 隔离的分享页** 数据不可混用；分享页仅展示 **该 token 绑定快照**。

---

## 5. 与实现的衔接说明

- **数据：** 分享载荷在结构上应能还原 `Task[]` 或等价 JSON，以便复用只读渲染与 `computeTaskStatistics`。
- **路由：** Next.js App Router 下新增 `src/app/share/[token]/page.tsx`（名称以最终实现为准）与上述 URL 一致。
- **验收：** 给定固定快照，分享页渲染树、状态、依赖文案与引擎规则一致；无效 token 不泄露信息；页面无编辑任务的主路径。

## 6. 待产品确认项与建议默认

若产品未另行指定，实现可按下列 **建议默认** 推进，减少阻塞：

| 议题 | 建议默认 | 说明 |
|------|----------|------|
| 链接过期 | **MVP 不强制过期**；若做托管链接，再增加「永不过期 / 7 天 / 30 天」 | 先保证创建与撤销链路；过期策略可迭代。 |
| 统计条 | **默认展示精简一行**（总任务数 + 完成数或比例），详情折叠或链到页内锚点 | 与 `stats` 全量卡片区分，避免分享页过长。 |
| 多语言 | **与主应用一致，英文 UI**；`title` 等用户内容原样展示 | 不单独为分享页做 locale 探测，除非主站已 i18n。 |

仍可由产品推翻以上默认。
